### 3. Task Scheduler (`5747a0a`)
- Nouveau `src/scheduler.ts` : système de tâches planifiées

  - Stockage JSON : `~/.mini-claw/tasks.json`
  - Polling toutes les 30 secondes
  - Parser cron minimal intégré (pas de dépendance externe)
  - Anti-drift : ancrage au temps prévu, pas à `Date.now()`
- Intégration dans `index.ts` : scheduler démarre après le preload des sessions
- Ajout de `runPiForScheduler()` dans `pi-runner.ts` : exécute un prompt sans typing indicators
- **Test fonctionnel** : Le bot a créé lui-même une tâche "Bonjour quotidien à 8h" en écrivant dans `tasks.json`

## Analyse de l'écosystème NanoClaw

- NanoClaw utilise SQLite + `cron-parser` + containers Docker pour le scheduler
- Notre approche est plus simple : JSON + polling + appel direct au Pi SDK
- Suffisant pour un usage mono-utilisateur Telegram

## Décisions

| Décision | Raison |
|----------|--------|
| JSON au lieu de SQLite pour les tâches | Simplicité — un seul utilisateur, pas besoin de concurrence |
| Parser cron minimal intégré | Évite une dépendance NPM pour 5 champs |
| Scheduler démarre après preload | Les sessions doivent être prêtes avant d'exécuter des tâches |
| Mémoire = fichiers plats | Le bot a déjà accès au filesystem via Pi SDK — pas besoin d'API spéciale |
