## Decisions
1. **Preload sans bloquer le bot**
   - **Choix**: `preloadSessions()` appelé sans `await`, en parallèle du `bot.start()`
   - **Rationale**: Le bot Telegram est accessible immédiatement (3.9s), les sessions se chargent en arrière-plan (12s)
   - **Alternative rejetée**: Attendre le preload avant de démarrer le bot — trop lent

2. **Guard contre la double création de session**
   - **Choix**: Map `sessionCreating` pour empêcher deux appels concurrents de créer la même session
   - **Rationale**: Le preload en arrière-plan peut courir avec le premier message Telegram

3. **Logger roulant plutôt que session-logger existant**
   - **Choix**: Logger dédié MiniClaw dans `~/.mini-claw/logs/`
   - **Rationale**: Le bot est un processus séparé de Pi, il a besoin de son propre fichier de log persistent

4. **PowerShell + VBS pour le tray plutôt qu'AHK ou Node.js**
   - **Choix**: PowerShell Windows Forms NotifyIcon lancé via VBS (fenêtre invisible)
   - **Rationale**: Natif Windows, pas de dépendance supplémentaire

5. **Auto-restart on crash**
   - **Choix**: Health timer 10s détecte process mort → restart automatique après 3s
   - **Rationale**: Le bot doit être résilient sans intervention humaine

## Problems
1. **node_modules manquants**
   - **Symptôme**: Le bot ne démarrait pas
   - **Cause**: Dossier node_modules pas commité et pas installé
   - **Solution**: `npm install`

2. **Erreurs TypeScript**
   - **Symptôme**: `InstanceType<typeof AuthStorage>` ne compile plus
   - **Cause**: Pi SDK a rendu le constructeur AuthStorage privé (factory `AuthStorage.create()`)
   - **Solution**: Remplacé par `AuthStorage` directement comme type

3. **Chemin avec espaces dans interactive_shell**
   - **Symptôme**: PowerShell ne trouvait pas le script `-File "C:\Users\julien\OneDrive\Coding\_Projets..."`
   - **Cause**: Git Bash escape les guillemets différemment de PowerShell
   - **Solution**: Créé `start.cmd` wrapper qui cd vers le bon dossier puis lance PowerShell

4. **WorkingDirectory invalide dans Process.Start()**
   - **Symptôme**: "Nom de répertoire non valide" 
   - **Cause**: `$PSScriptRoot` + `Split-Path -Parent` donnait un chemin relatif non résolu
   - **Solution**: `Resolve-Path` pour obtenir le chemin absolu, + `Get-Command node` pour résoudre node.exe

5. **Icônes fantômes dans le tray Windows**
   - **Symptôme**: 2 icônes visibles après kill d'un process
   - **Cause**: Bug connu Windows — icônes non nettoyées tant que non survolées
   - **Solution**: Passer la souris dessus, l'icône fantôme disparaît

6. **Git push refusé**
   - **Symptôme**: Permission denied to `theflysurfer` sur `JK10gming/mini-claw.git`
   - **Cause**: Remote origin pointe vers un repo tiers
   - **Solution**: En attente — user doit configurer son propre remote

## Code Patterns
### Architecture
- **Preload pattern**: Sessions lourdes pré-chargées au boot en arrière-plan
- **Creation guard**: Map de promises en cours pour éviter les races
- **Timer class**: Mesure lap-by-lap avec log automatique au `.done()`

### Style
- Tagged logging: `[tag]` prefix sur chaque ligne (boot, pi-sdk, prompt, msg, tool)
- Timers structurés: `⏱ START ... / ├─ lap / ⏱ DONE total [breakdown]`
